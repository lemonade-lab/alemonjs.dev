# 更新记录

## 支持 hooks 模式

```ts title="src/response/**/*/res.ts"
import { useMessage, Format } from 'alemonjs'
export default async event => {
  // hook 的第一个参数为 event，在 response 中会自动劫持event上下文
  const [message] = useMessage()

  const format = Format.create().addText('hello word')
  message.send({ format })
}
```

## `useEvent`

安全的读取 event 和 next，handler 无需声明参数

```ts
import { useEvent } from 'alemonjs'
export default async () => {
  const [event, next] = useEvent({
    selects: ['message.create'],
    regular: /hello/ // 可选，正则匹配
  })
  // 匹配结果为布尔值
  if (!event.match.selects || !event.match.regular) {
    next()
    return
  }
  // 匹配成功，通过 event.current 访问事件对象
  // 通过 event.value 访问原始数据
}
```

## FormatEvent 链式构建器

平台适配器现在统一使用 `FormatEvent` 链式构建器创建事件对象，根据事件名自动约束可用方法。

```ts
import { FormatEvent } from 'alemonjs'

// message.create 拥有全部字段方法
const msg = FormatEvent.create('message.create')
  .addPlatform({ Platform: 'discord', value: raw, BotId: botId })
  .addGuild({ GuildId: 'g1', SpaceId: 's1' })
  .addChannel({ ChannelId: 'c1' })
  .addUser({ UserId: 'u1', UserKey: 'k1', IsMaster: false, IsBot: false })
  .addMessage({ MessageId: 'm1' })
  .addText({ MessageText: 'hello' })
  .addMedia({ MessageMedia: [{ Type: 'image', Url: '...' }] })
  .addOpen({ OpenId: 'o1' }).value

// guild.join 只有 Platform/Guild/Channel/User/Message
const join = FormatEvent.create('guild.join')
  .addPlatform({ Platform: 'qq', value: raw, BotId: botId })
  .addGuild({ GuildId: 'g1', SpaceId: 's1' })
  .addChannel({ ChannelId: 'c1' })
  .addUser({ UserId: 'u1', UserKey: 'k1', IsMaster: false, IsBot: false })
  .addMessage({ MessageId: '' }).value
// join.addText → 类型错误 ✓
```

## 新增事件类型

事件系统扩展至 **27 种**，新增以下事件：

| 事件名                  | 说明                 |
| ----------------------- | -------------------- |
| `message.pin`           | 消息置顶             |
| `channel.update`        | 频道更新             |
| `guild.update`          | 服务器/群组更新      |
| `member.ban`            | 成员封禁             |
| `member.unban`          | 成员解封             |
| `member.update`         | 成员信息更新         |
| `notice.create`         | 公共通知（戳一戳等） |
| `private.notice.create` | 私有通知             |
| `private.friend.remove` | 好友移除             |

所有平台适配器的 hook MAP 已同步覆盖全部 27 种事件。

## EventBuilder 类型安全增强

`EventBuilder<T>.value` 的类型从 `Partial<Events[T]>` 改为 `Events[T]`，确保类型系统中事件对象始终是完整的。

## 安全加固

### 路径遍历防护

接口路由新增 `safePath()` 安全函数，所有 `/app/*` 和 `/apps/:app/*` 路由的文件路径均经过校验，防止 `../` 路径遍历攻击。

```ts
// resolve 后确认路径仍在 root 目录内
safePath(root, untrustedPath) // 越界返回 null → 403
```

### 包名注入防护

`/apps/:app/*` 路由新增 `isValidPackageName()` 校验，阻止通过非法包名（如 `../../etc`）进行路径注入。

```ts
isValidPackageName('my-app') // true
isValidPackageName('@scope/my-app') // true
isValidPackageName('../../etc') // false → 400
```
